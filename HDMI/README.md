# Документация проекта: Генератор HDMI (640x480 @ 60Hz)

## Обзор архитектуры
Проект предназначен для инициализации HDMI-трансмиттера (ADV7513) по шине I2C и последующей генерации видеосигнала стандарта VGA (разрешение 640x480 при 60 кадрах в секунду), который конвертируется чипом в HDMI. 

Проект состоит из нескольких логических блоков:
1. **Генерация тактовой частоты (PLL)**: Из входных 50 МГц создается пиксельная частота 25.175 МГц (в коде `clk_25`).
2. **Инициализация HDMI-чипа**: Модули `hdmi_config` и `i2c_simple_controller` настраивают внутренние регистры ADV7513 при старте.
3. **Генерация развертки (Тайминги)**: Модуль `vga_generator` (и альтернативный ему `hdmi_sync`) отвечает за создание синхроимпульсов (H-Sync, V-Sync) и координат текущего пикселя (X, Y).
4. **Генерация изображения**: Логика внутри `hdmi_top` (или отдельный модуль `pattern_gen`) закрашивает пиксели нужным цветом в зависимости от их координат X и Y.

## Описание модулей

### 1. `hdmi_top` (Top-Level Модуль)
Главный модуль, объединяющий все компоненты. 
* **Что делает:** Принимает внешние сигналы (кнопки, клок 50 МГц), управляет светодиодами для диагностики (LED 0 светится, если PLL запущен; LED 1 — если I2C конфигурация завершена). Инстанциирует PLL, конфигуратор I2C и генератор таймингов. Также здесь прописана логика раскраски (вывод красного, зеленого и синего цветов в зависимости от координаты X).
* **Как редактировать:** Если ты захочешь выводить изображение из памяти (BRAM) или изменить узор, тебе нужно будет заменить блок `always_comb` (где назначается `HDMI_TX_D`) на свой собственный или подключить сюда модуль `pattern_gen`.

### 2. `vga_generator` и `hdmi_sync`
Оба этих модуля выполняют одну и ту же задачу — генерируют видео-тайминги. В твоем `hdmi_top` сейчас используется `vga_generator`. 
* **`vga_generator`:** Хардкодная реализация таймингов для 640x480.
* **`hdmi_sync`:** Более грамотная, параметризованная версия. Она использует параметры (`H_FRONT_PORCH`, `H_SYNC_PULSE` и т.д.), что делает ее универсальной.
* **Как редактировать:** Если захочешь поменять разрешение (например, на 720p), лучше переключить `hdmi_top` на использование `hdmi_sync` и изменить значения `localparam` в нем на тайминги для 720p (и не забыть перенастроить PLL на 74.25 МГц).

### 3. `hdmi_config`
Конечный автомат (FSM), который хранит массив настроек для ADV7513.
* **Что делает:** По очереди берет пары "Адрес регистра + Данные" из блока `case (rom_addr)` и скармливает их I2C-контроллеру. Настраивает чип на прием 24-битного RGB 4:4:4.
* **Как редактировать:** Если в даташите на ADV7513 ты найдешь настройки, которые захочешь изменить (например, поменять цветовое пространство, включить аудио, изменить формат входных пинов), тебе нужно добавить или изменить строки в `case (rom_addr)`.

### 4. `i2c_simple_controller`
Модуль физического уровня шины I2C (Bit-banging).
* **Что делает:** Делит 50 МГц частоту, чтобы получить нужную для I2C скорость (около 100 кГц). Реализует сложный автомат состояний, который дергает пины `SCL` и `SDA` по спецификации протокола I2C (Start -> Device Addr -> ACK -> Reg Addr -> ACK -> Data -> ACK -> Stop).
* **Как работать:** Этот модуль полностью универсален. Его можно не трогать, он просто надежно выполняет отправку данных, которые ему передает `hdmi_config`.

### 5. `pattern_gen`
Отдельный генератор паттерна.
* **Что делает:** Получает координаты и отдает 24-битный цвет RGB. В текущей реализации `hdmi_top` он не инстанцирован (вместо него написан `always_comb`), но его удобно использовать для выноса логики отрисовки в отдельный файл.

## Инструкция по модификации (How-To)

### Как изменить цвета или нарисовать что-то другое?
Найди этот блок в `hdmi_top`:
```systemverilog
    always_comb begin
        if (vga_de) begin
            if (x < 213)      HDMI_TX_D = 24'hFF0000; // Красный
            else if (x < 426) HDMI_TX_D = 24'h00FF00; // Зеленый
            else              HDMI_TX_D = 24'h0000FF; // Синий
        end else begin
            HDMI_TX_D = 24'h000000;
        end
    end
